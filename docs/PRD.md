# PRD: Pulsar — веб-обёртка для управления Asterisk контакт-центром

## 1. Overview

**Pulsar** — веб-приложение на Ruby on Rails для управления телефонией на базе Asterisk. Продукт позволяет быстро разворачивать и настраивать инстансы контакт-центра в разных регионах (США, ОАЭ, Украина и др.) через веб-интерфейс, без ручной правки конфигов Asterisk.

**Стадия:** MVP
**Первый регион:** 1 инстанс, 10 операторов, до 30 одновременных звонков

---

## 2. Сегменты и работы (AJTBD)

### Сегмент 1: Администратор-развертыватель

**Core Jobs:**
- **Когда** компания открывает КЦ в новом регионе, **хочу** развернуть инстанс телефонии за < 1 день без глубоких знаний Asterisk
- **Когда** нужно изменить правила маршрутизации или добавить очередь, **хочу** сделать это через веб-интерфейс, а не через конфиг-файлы
- **Когда** что-то ломается, **хочу** быстро диагностировать проблему через веб-панель

**Big Job:** Обеспечить стабильную телефонию во всех регионах присутствия, чтобы бизнес мог масштабироваться без роста технической команды.

### Сегмент 2: Супервизор контакт-центра

**Core Jobs:**
- **Когда** идёт рабочая смена, **хочу** видеть статусы операторов, длину очередей и SLA в реальном времени
- **Когда** оператор долго на линии или клиент ждёт, **хочу** быстро вмешаться — перераспределить нагрузку
- **Когда** нужно оценить качество работы, **хочу** прослушать записи и посмотреть статистику

**Big Job:** Держать качество обслуживания на целевом уровне, чтобы КЦ выполнял KPI.

### Сегмент 3: Оператор контакт-центра

**Core Jobs:**
- **Когда** поступает входящий звонок, **хочу** принять его через софтфон с минимальной задержкой
- **Когда** не могу решить вопрос сам, **хочу** переключить звонок на другого специалиста или очередь
- **Когда** заканчиваю смену, **хочу** корректно выйти из очереди

**Big Job:** Обрабатывать звонки быстро и без ошибок, чтобы выполнять нормативы.

## 3. Рискованное предположение, валидируемое через MVP

**Предположение:** Asterisk + Rails-обёртка обеспечит стабильную работу при 30 одновременных звонках с real-time мониторингом и записью, а веб-админка реально ускорит развёртывание и настройку по сравнению с ручной конфигурацией.

**Категория:** операционные/технологические

**Score: P3 × I5 = 15**

**Методы проверки:**
- Нагрузочное тестирование: 30 одновременных SIP-звонков через SIPp
- Proof of concept: AMI listener + Action Cable при потоке событий
- Замер времени развёртывания нового инстанса (цель: < 1 день)

## 4. MVP — обязательный функционал

### 4.1 Управление операторами
- CRUD операторов (имя, SIP-аккаунт, навыки)
- Назначение оператора в одну или несколько очередей
- Отображение статуса оператора: online / offline / busy / on break
- Управление статусом через веб-интерфейс (пауза, выход из очереди)

### 4.2 Управление очередями
- CRUD очередей
- Настройка стратегии распределения (ringall, leastrecent, fewestcalls, random, rrmemory)
- Настройка таймаута ожидания в очереди (конфигурируемый)
- Настройка действия при превышении таймаута: голосовое сообщение / сброс / перенаправление на другую очередь
- Добавление/удаление операторов в очередь

### 4.3 Маршрутизация входящих звонков
- Правила маршрутизации: входящий номер → очередь
- Веб-интерфейс для создания и редактирования правил
- Применение правил без перезагрузки Asterisk (через ARI/AMI)

### 4.4 Обработка отключений оператора
- При потере связи оператора во время звонка — автоматическое перенаправление звонка обратно в очередь другому оператору
- Автоматическое изменение статуса оператора на offline

### 4.5 Запись разговоров
- Автоматическая запись всех разговоров
- Хранение записей локально на сервере инстанса (архитектура готова к миграции на S3)
- Бессрочное хранение записей
- Веб-интерфейс для поиска записей (по дате, оператору, номеру телефона)
- Встроенный аудиоплеер для прослушивания

### 4.6 Базовая статистика и мониторинг
- Дашборд: активные звонки, операторы online, загрузка очередей
- Real-time обновление статусов (Action Cable, задержка < 2 сек)
- Метрики: количество звонков, пропущенные (abandoned), среднее время ожидания, среднее время разговора
- Фильтрация по дате и очереди

### 4.7 Ролевая модель доступа
- **Администратор:** полный доступ — настройка очередей, маршрутизации, управление пользователями, просмотр всей статистики
- **Супервизор:** мониторинг в реальном времени, статистика, прослушивание записей, управление операторами в своих очередях
- **Оператор:** управление своим статусом (online/pause/offline), просмотр своей статистики

### 4.8 Мультиязычность
- Интерфейс на английском языке по умолчанию
- Rails I18n для поддержки переключения языков (EN, RU, UA) в следующих фазах

### 4.9 Развёртывание
- Docker Compose: Rails + Asterisk + PostgreSQL (Redis не нужен — SolidQueue, SolidCable, SolidCache работают через БД)
- Конфигурация через переменные окружения (.env по регионам)
- Цель: развернуть новый инстанс КЦ за < 1 день

## 6. Технический стек

| Компонент | Технология |
|-----------|-----------|
| Framework | Ruby on Rails 8 (Hotwire) |
| Asterisk интеграция | ARI (REST) + AMI (TCP) |
| Real-time | Action Cable (SolidCable — без Redis) |
| База данных | SQLite3 (+ отдельные БД для cache, queue, cable в production) |
| Фоновые задачи | SolidQueue (встроен в Rails 8, без Redis) |
| Кеширование | SolidCache |
| Фронт | Hotwire (Turbo Frames/Streams) + Tailwind CSS |
| Аутентификация | rails g authentication (встроенная в Rails 8) |
| Авторизация | Pundit (роли: admin, supervisor, agent) |
| Инфраструктура | Docker + Kamal |

## 7. Инфраструктура MVP (VPS)

### 7.1 Компоненты на VPS

| Компонент | Назначение |
|-----------|-----------|
| Asterisk | Телефония — SIP, очереди, записи разговоров |
| SQLite3 | БД для Rails (CDR, конфиги, пользователи) — встроена, отдельный сервис не нужен |
| Rails-приложение (Pulsar) | Веб-админка и API |
| Nginx | Reverse proxy, SSL-терминация, отдача статики и записей |

### 7.2 Внешние зависимости

| Компонент | Назначение | Статус |
|-----------|-----------|--------|
| SIP-транк провайдер | Подключение к телефонной сети (входящие/исходящие звонки) | Будет выбран позже |
| DID-номера | Телефонные номера для входящих звонков (покупаются у SIP-провайдера) | Будут приобретены после выбора провайдера |
| Доменное имя + SSL | Для веб-интерфейса (Let's Encrypt) | Требуется настроить |

### 7.3 Требования к VPS

- **ОС:** Debian 12 или Ubuntu 22.04 LTS
- **Ресурсы (10 операторов / 30 одновременных звонков):** 2 vCPU, 4 GB RAM, 80+ GB SSD
- **Порты:** SIP (5060 UDP), RTP (10000–20000 UDP), HTTPS (443), SSH (22)

## 8. Edge cases

| Ситуация | Поведение |
|----------|----------|
| Оператор отключился во время звонка | Звонок автоматически возвращается в очередь и назначается другому оператору |
| Все операторы заняты | Клиент ждёт в очереди; при превышении настроенного таймаута — выполняется настроенное действие (голосовое сообщение / сброс / переадресация) |
| Asterisk инстанс недоступен | Веб-интерфейс показывает статус "disconnected", администратор получает уведомление |
| Диск заполнен записями | Уведомление администратору при достижении порога (напр. 80% диска) |
| Оператор назначен в несколько очередей | Звонок поступает из очереди с наивысшим приоритетом; оператор помечается busy для всех очередей |
